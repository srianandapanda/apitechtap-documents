AI TechTap Core - Full API Contract

End-to-end API product contract for the complete repository: Auth, User, Plan Admin, Payments, and AI Career Advisor modules.

OpenAPI 3.x Spring Boot 3 Java 21 Version 1.0

Product Purpose

  • Onboard users with email/phone or Google token.
  • Capture dynamic profile fields across user groups.
  • Support plan purchase and payment lifecycle.
  • Deliver AI-powered career conversation and suggestions.

Base Access

  • Base URL: http://localhost:8081
  • Swagger UI: /swagger-ui/index.html
  • OpenAPI JSON: /v3/api-docs

Auth Rules

  • Bearer JWT: user, payments, AI APIs.
  • Basic Auth: plan admin create/update.
  • Public: auth APIs, payment callback/webhook, docs.

API Index

  • Auth APIs

Auth APIs

POST/api/auth/signup

Create user credential identity with email or phone plus password.

200409500

Request sample

{
  "email": "priya.student@example.com",
  "phone": "9876543210",
  "password": "StrongPass@123"
}

Success response sample

{
  "userId": "4b40e5fd-4194-4f11-ac58-ea0ea24f640c",
  "message": "User created successfully"
}

POST/api/auth/login

Login with email or mobile_number and password. Returns JWT token.

200401

Request sample

{
  "email": "priya.student@example.com",
  "password": "StrongPass@123"
}

Success response sample

{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "expiresIn": 3600
}

POST/api/auth/logout

Logout user by revoking bearer token from Authorization header.

200

Response sample

{ "message": "User logged out successfully" }

POST/api/auth/forgot-password

Request password reset (no account enumeration). See docs/API-CONTRACTS.md. Implement with password_reset_tokens table.

200

Request sample

{ "email": "user@example.com" }

POST/api/auth/reset-password

Set new password using one-time token from email/SMS.

200400401

Request sample

{ "token": "opaque-reset-token", "newPassword": "NewSecurePass@1" }

Removed: Google signup is not supported for now.

User APIs

GET/api/users/{userId}

Get current user profile. Path userId must match JWT user.

200401403404

POST/api/users (Dynamic)

With JWT: update user details only (dynamic profile). New accounts: use POST /api/auth/signup first. Supports varied fields for students and professionals; additional keys are allowed.

200

200404

200401

AI APIs

Fetch current user conversation summary and career suggestions. JWT required.

200401

Data Storage Assurance

  • Orders and payments are persisted in PostgreSQL tables: orders, payments.
  • Payment module uses JPA entities/repositories, not MongoDB documents.
